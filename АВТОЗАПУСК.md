# Надёжный автозапуск (без ручных кнопок)

Родное расписание GitHub Actions (`on: schedule`) часто срабатывает с большой
задержкой или пропускает запуск. Чтобы посты выходили строго по времени, запуск
дёргает **внешний планировщик** через API GitHub (событие `workflow_dispatch`).

Все наши workflow уже умеют запускаться вручную (`workflow_dispatch`), так что
внешнему планировщику остаётся просто «нажимать кнопку» по расписанию.

## Шаг 1. Создать токен GitHub (PAT)

1. GitHub → Settings (профиля) → Developer settings → **Personal access tokens** →
   **Fine-grained tokens** → Generate new token.
2. **Repository access**: Only select repositories → `nkg-digest`.
3. **Permissions** → Repository permissions → **Actions: Read and write**.
4. Срок (Expiration) — на год. Сгенерировать, **скопировать токен** (вид `github_pat_...`).

## Шаг 2. Завести задания на cron-job.org

Зарегистрируйтесь на https://cron-job.org (бесплатно). Для каждого запуска создайте
Cronjob с одинаковыми настройками запроса и своим временем.

**Общие настройки запроса (для всех заданий):**

- **URL**: `https://api.github.com/repos/salkutsan-collab/nkg-digest/actions/workflows/<ФАЙЛ>/dispatches`
  (подставьте имя файла workflow из таблицы ниже)
- **Method**: `POST`
- **Headers**:
  - `Authorization: Bearer <ВАШ_PAT>`
  - `Accept: application/vnd.github+json`
  - `X-GitHub-Api-Version: 2022-11-28`
- **Request body**: `{"ref":"master"}`
- В настройках задания выберите **часовой пояс Europe/Moscow**, тогда время вводите московское.

**Расписание и файлы (время московское):**

| Время | День | Workflow (файл) | Что |
|---|---|---|---|
| 08:00 | каждый день | `daily.yml` | Дайджест дня в каналы |
| 14:00 | каждый день | `feature.yml` | «Разбор дня» в каналы |
| 18:00 | каждый день | `preview.yml` | Сбор на завтра + черновики в личку |
| 07:00 | воскресенье | `weekly.yml` | Обновление базы |
| 09:00 | понедельник | `comics.yml` | Комиксы в личку |

Итого 5 заданий. Каждое — POST на свой `<ФАЙЛ>` в нужное время.

## Проверка

После создания задания нажмите в cron-job.org «Test run» (или дождитесь времени).
Успех — ответ GitHub **HTTP 204** (без тела). В GitHub → Actions появится запуск
соответствующего workflow. Если приходит 401/403 — проверьте токен и право
**Actions: Read and write**.

## Заметки

- Токен PAT даёт право запускать наши workflow. Храните его только в cron-job.org;
  при утечке отзовите в GitHub и создайте новый.
- Родное `schedule` в workflow можно оставить как запасной (хуже-чем-ничего) или убрать.
- Секреты репозитория (ключи YandexGPT/OpenAI/Telegram/Max) для самих запусков
  по-прежнему берутся из GitHub Secrets — внешний планировщик их не касается.
