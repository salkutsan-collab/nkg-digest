# Дайджест НКГ - события культуры Санкт-Петербурга

Автоматический дайджест событий участников «Новой культурной географии»:
дизайн, современное искусство, стрит-арт, креативные пространства.
Раз в день - анонс ближайшей недели, раз в неделю - большой обзор.

## Как это устроено (простыми словами)

**База участников** - файл [`data/participants.yaml`](data/participants.yaml).
Список мест с адресами и ссылками, откуда брать новости (сайт, ВКонтакте,
Telegram), и страницей афиши `events_url`.

**Агент 1 - «Реестр»** ([`agents/agent1_registry.py`](agents/agent1_registry.py)).
Проверяет ссылки, находит на сайтах афиши и соцсети, дополняет базу.
Читает «входящие» [`data/inbox.md`](data/inbox.md) - правки от человека.

**Агент 2 - «Дайджест»** ([`agents/agent2_digest.py`](agents/agent2_digest.py)).
Заходит на страницы афиш, моделью (GigaChat) достает события на неделю
(что, когда, где), собирает дайджест и кладет в `digests/`.
Отправка в Telegram - следующий шаг.

**Модель** - через [`agents/llm.py`](agents/llm.py). По умолчанию GigaChat
(оплата в рублях), запасной вариант - Anthropic. Ключи берутся из `.env`.

## Где это работает, когда компьютер выключен

Конвейер крутится в облаке на GitHub Actions по расписанию (бесплатно).
База - в репозитории, ключи - в секретах GitHub. (Настроим на шаге 5.)

## Как запустить руками (для проверки)

```
py -m pip install -r requirements.txt

# агент 1: проверить базу / внести правки
py agents/agent1_registry.py
py agents/agent1_registry.py --apply

# агент 2: посмотреть формат без ключа и сети
py agents/agent2_digest.py --self-test
# дайджест со ссылками на афиши (без модели)
py agents/agent2_digest.py --no-llm
# полный сбор моделью (нужен ключ GigaChat в .env)
py agents/agent2_digest.py
```

## Ключ модели (YandexGPT)

1. Войти в Yandex Cloud: console.yandex.cloud (по Яндекс-аккаунту).
2. Создать платежный аккаунт (оплата в рублях; для новых есть грант).
3. Открыть каталог (folder) и скопировать его идентификатор - строка вида `b1g...`.
4. Каталог -> Сервисные аккаунты -> создать аккаунт, дать роль `ai.languageModels.user`.
5. Открыть этот сервисный аккаунт -> создать ключ -> **API-ключ**, скопировать секрет.
6. Скопировать `.env.example` в `.env` и вписать:
   `YANDEX_API_KEY=...` и `YANDEX_FOLDER_ID=...`.

Запасной вариант - GigaChat: `LLM_PROVIDER=gigachat` и `GIGACHAT_CREDENTIALS=...`.
Файл `.env` в репозиторий не попадает (защищен `.gitignore`).

## Telegram (бот и канал)

Как завести (один раз):

1. В Telegram открыть **@BotFather** -> `/newbot` -> задать имя и username.
   BotFather выдаст **токен** - вписать в `.env` как `TELEGRAM_BOT_TOKEN`.
2. Создать **канал** (проще публичный, с адресом вида `@nkg_afisha`).
3. В настройках канала -> Администраторы -> добавить вашего бота,
   дать право «Публикация сообщений».
4. Указать `TELEGRAM_CHAT_ID` в `.env`:
   - публичный канал: `@nkg_afisha`;
   - закрытый канал: числовой `-100...` (узнать командой `--updates`,
     предварительно опубликовав в канале любое сообщение).
5. Проверить связь и отправить дайджест:

```
py agents/notify_telegram.py --test          # должно прийти сообщение в канал
py agents/notify_telegram.py --updates       # показать chat_id виденных чатов
py agents/agent2_digest.py --no-llm --send   # собрать и сразу отправить
```
